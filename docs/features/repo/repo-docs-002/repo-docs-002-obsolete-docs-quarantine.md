# repo-docs-002: obsolete docs quarantine

## Summary
Keep active documentation focused on the current release and move historical or superseded materials under `docs/obsolete/`.

## Behavior
- `docs/` keeps only current release control files, active feature docs, active task dossiers, current guides, and current release notes.
- Historical release matrices, older task drafts, superseded feature docs, and old narrative docs move under `docs/obsolete/`.
- Historical material is preserved for lookup, but no longer competes with active contributor guidance.

## Notes
- `docs/v1-2/` remains the source of truth for the current approved release.
- `docs/obsolete/` mirrors prior structure where practical so old references are still understandable.
