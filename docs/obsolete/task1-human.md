# OptiDev --- MVP Specification

**CLI tool:** `optid`\
**Goal:** Local orchestration layer for AI coding agents (Codex, Claude
Code, OpenCode, etc.) with a project‑aware workspace.

------------------------------------------------------------------------

# 1. Concept

OptiDev is a **local AI development orchestrator** that prepares and
restores an agent workspace for a project.

Instead of manually starting terminals, agents, logs, tests and
environments, a single command restores everything:

``` bash
optid <project>
```

Example:

``` bash
optid opticlaw
```

This command:

1.  Opens a **Zellij workspace**
2.  Loads project context
3.  Restores agent history
4.  Starts development services
5.  Launches agent runners

The user immediately continues work.

------------------------------------------------------------------------

# 2. Core Workflow

### Start session

``` bash
optid <project>
```

Example:

``` bash
optid opticlaw
```

System actions:

1.  Locate project
2.  Load project configuration
3.  Restore last session state
4.  Start development environment
5.  Launch Zellij layout
6.  Start agents
7.  Restore conversation context

Terminal shows:

    OptiDev workspace ready.
    What are we doing today?

# 3. Workspace Layout (Zellij)

Default MVP layout:

    ┌───────────────┬───────────────┐
    │ planner agent │ coder agent   │
    ├───────────────┼───────────────┤
    │ tests         │ logs          │
    └───────────────┴───────────────┘

Panels:

  Pane      Purpose
  --------- ------------------
  planner   task planning
  coder     code generation
  tests     test runner
  logs      application logs

Layout defined dynamically via:

    layout.kdl

Generated by OptiDev.

Example:

    .project/
      agents/
          planner.md
          coder.md
          tester.md

      skills/
          backend.md
          frontend.md

      mcp/
          tools.yaml

------------------------------------------------------------------------

# 6. Agents Configuration

Agents defined via Markdown files.

Example:

    .project/agents/planner.md

    Agent: planner

    Role:
    Plan development tasks.

    Responsibilities:
    - break down features
    - track tasks
    - maintain roadmap

    Skills:
    - architecture
    - planning

Example coder agent:

    Agent: coder

    Role:
    Implement code changes.

    Skills:
    - backend
    - refactoring
    - testing

------------------------------------------------------------------------

# 7. Skills Configuration

Skills describe reusable capabilities.

Example:

    .project/skills/backend.md

    Skill: backend

    Stack:
    - Python
    - FastAPI
    - Postgres

    Guidelines:
    - write tests
    - follow clean architecture

Skills are injected into agent context.

------------------------------------------------------------------------

# 8. Context System

Context sources:

    AGENTS.md
    SKILLS.md
    repo structure
    recent history
    project metadata

Purpose:

Build a **structured context payload** for each agent session.

Example pipeline:

    repo scan
    ↓
    load agents
    ↓
    load skills
    ↓
    load memory
    ↓
    construct prompt context

------------------------------------------------------------------------

# 9. Agent Runner

Unified interface for AI agents.

Supported runners:

-   Codex
-   Claude Code
-   OpenCode (future)

Example interface:

``` python
agent.run(task)
agent.resume(session_id)
agent.stop()
```

Runner implementation lives in:

    optidev/runners/

Example:

    runners/
      codex.py
      claude.py

------------------------------------------------------------------------

# 10. Memory System

Memory stored locally.

Storage:

    .optidev/
      sessions/
      memory/

Storage format:

SQLite database.

Purpose:

-   restore conversations
-   persist decisions
-   store task history

Example schema:

    sessions
    messages
    tasks
    decisions

------------------------------------------------------------------------

# 11. Project Discovery

Projects stored globally.

Example directory:

    ~/.optidev/projects

Command:

    optid projects

Lists available workspaces.

Projects can also be discovered by scanning:

    ~/dev
    ~/projects

------------------------------------------------------------------------

# 12. Dev Environment Hooks

Projects may define environment hooks.

Example:

    .project/config.yaml

    dev:
      start:
        - docker compose up
        - npm run dev

    tests:
      watch:
        - pytest

OptiDev executes these commands when workspace starts.

------------------------------------------------------------------------

# 13. Test Runner

Tests run in dedicated pane.

Supported patterns:

    pytest
    npm test
    cargo test

Configurable per project.

Example:

    tests:
      command: pytest -q
      watch: true

------------------------------------------------------------------------

# 14. Logs Pane

Logs show application output.

Example sources:

    docker logs
    npm dev logs
    python server logs

Configured in:

    config.yaml

------------------------------------------------------------------------

# 15. Plugin System (MVP Minimal)

Plugin directory:

    optidev/plugins/

Plugin interface:

``` python
class Plugin:

    def on_workspace_start(self):
        pass

    def on_agent_message(self, message):
        pass

    def on_workspace_stop(self):
        pass
```

Plugins loaded dynamically.

------------------------------------------------------------------------

# 16. Plugins
  1. Telegram Plugin (Extension)

Telegram plugin enables remote interaction.

Capabilities:

    continue conversations
    trigger commands
    monitor agents

Example commands:

    /status
    /tests
    /logs
    /stop

Plugin runs optional bot.

  2. Skills search (Extension)

Search and install skills using skills.sh

Capabilities:

    find skill
    install in current project (in proper path)
    allow agents to use it

Example commands:

    /skills search ...
    /skills install ...

3. same for mcp https://mcpmarket.com/server 
4. same for agents https://aiagentslist.com/

# 20. MVP Scope

-   CLI (`optid`)

-   agent runner abstraction
-   agent configuration
-   skills configuration
-   SQLite memory
-   dev environment hooks
-   tests pane
-   logs pane
-   basic plugin system

Not included in MVP:

-   advanced agent swarm coordination
-   vector databases
-   metrics dashboards
-   distributed agents
-   skills.sh skills search

-   agents search

These are future extensions.
