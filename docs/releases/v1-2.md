# Release v1-2

## Scope
- Introduce Agent Memory Graph as structured project memory on top of OptiDev runtime.
- Store tasks, features, releases, decisions, open loops, and sessions in a replaceable graph storage module.
- Surface structured restore summaries and memory lookup commands.

## Delivered
- `ui/apps/server/src/optidevMemory.ts` provides native TS/Bun memory ingestion, release linking, typed lookup, and digest rendering.
- `optid memory` exposes summary, feature, task, release, and open-loop lookups through the native Bun CLI.
- workspace start and resume print and propagate structured project memory into runner context and generated startup artifacts.
- Telegram transport is consolidated into the native integration layer and covered by colocated server/browser tests.
- runner command resolution is owned by the native startup layer and persisted in `runner.json`.
- repository implementation reports now live under `tasks-log/`.
